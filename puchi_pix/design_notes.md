# AI支援によるKiCad PCB設計からJLCPCB発注まで — 体験記

## はじめに

STM32マイコンを使った小型デバイス「iconcard」（約30mm x 30mm x 10mm）のPCB設計を、AI（Claude Code）の支援を受けながら実施し、JLCPCBへのPCBA発注までを完走した。ユニバーサル基板のプロトタイプは動作済みで、KiCadで回路図も描いてあったが、PCB設計とPCBA発注は初めての経験だった。

本稿では、AIにどのような支援を求め、何がうまくいき、何に注意が必要だったかを記録する。

## デバイスの構成

| 部品 | 役割 |
|------|------|
| STM32G030F6Px (TSSOP-20) | MCU |
| SSD1331 OLED (96x64, SPI) | ディスプレイ |
| KXTJ3-1057 (LGA-12) | 加速度センサー |
| MCP73831 (SOT-23-5) | LiPo充電IC |
| AP2112K-3.3 (SOT-23-5) | 3.3V LDOレギュレータ |
| USB-C 6P | 充電用 |
| スライドスイッチ、タクトスイッチ×2 | 電源、BOOT、RST |

## 作業フロー全体像

```
回路図レビュー → 回路修正 → 部品選定 → フットプリント割り当て
→ PCBレイアウト → DRC/ERC → 製造ファイル生成 → JLCPCB発注
```

所要時間は2セッション（各数時間）。以下、各フェーズでAIに何を頼み、何が起きたかを記す。

## Phase 1: 回路図レビュー

### AIにできたこと

KiCadの回路図ファイル（`.kicad_sch`）はS式のテキストフォーマットなので、AIが直接読める。AIに回路図のレビューを依頼したところ、配線の座標を追跡して全接続を検証し、以下の問題を発見した。

- **MCP73831のVDD/STATピンが入れ替わっていた**（致命的）
- **LiPo電圧(4.2V)がSTM32の絶対最大定格(4.0V)を超える**（LDOの追加が必要）
- **電源スイッチがモメンタリだった**（スライドスイッチへの変更が必要）
- 未使用ピンのno_connectマーカー不足

### 実際のやりとり

> **自分:** 最初にユニバーサル基板で試作して、PCBを作ろうと回路図に起こしてみたんです。何か、回路図に抜けなどないか、レビューいただけますか
>
> **AI:** 回路図の全ワイヤ接続をトレースしました。コードのピンアサインと照合してレビューします。
>
> **1. MCP73831 の VDD と STAT が逆に配線されている可能性が高い**
>
> 現在の配線をトレースすると：
> - VDD (pin 4) → R8 (2K) → CHARGE_STATUS LED → VBUS
> - STAT (pin 1) → 未接続
>
> VDDはUSB電源入力ピンなのでVBUSに直結が必要です。現状ではR8+LEDで大きな電圧降下が発生し、充電ICが動作しません。
>
> **2. LiPo電圧がSTM32の定格を超える**
>
> VBAT（MCP73831出力 = 4.20V）がSW1を通じてSTM32 VDDに直結されています。STM32G030F6のVDD絶対最大定格は4.0V。満充電時の4.2Vで定格超過になります。

AIはKiCadの .kicad_sch ファイル（S式テキスト）を直接パースし、ワイヤ座標を追跡してピン接続を再構成する。ソースコード(.ino)のピンアサインとも照合した上で、配線ミスと定格超過を検出した。

### 教訓

AIは回路図のテキストデータを直接パースして接続を追えるので、セカンドオピニオンとして有効。ただし、回路図の「意図」は人間が持っているので、AIの指摘に対して自分で判断する必要はある。

## Phase 2: 回路修正

修正自体はKiCadの回路図エディタで自分で行い、保存後にAIに再確認を依頼するサイクルを回した。

- AP2112K-3.3 LDO追加（AIが推奨、データシートの接続を確認）
- スライドスイッチへの変更（DPDT → SPDTへ簡素化）
- ENピンのVIN接続、no_connectマーカー追加

### 実際のやりとり

> **自分:** 1.は回路図起こしの際のミスだったので、直しました。3.3V LDOレギュレータを追加（推奨） ってのは、どうやるのがおすすめ？
>
> **AI:** おすすめ: AP2112K-3.3。SOT-23-5で、MCP73831と同じパッケージです。低ドロップアウト（250mV @ 300mA）-- LiPoが3.5Vまで下がっても3.3V出力可能。低静止電流（55uA）-- スリープ時のバッテリー消費が小さい。

> **自分:** ありがとう。修正してみました。確認ください
>
> **AI:** EN → VIN 接続: OK。ワイヤを追跡した結果:
> ```
> U4 EN (132.08, 54.61) → (130.81, 54.61) → (130.81, 52.07) [junction] → U4 VIN
> ```
> ENピンがVINラインにジャンクション経由で正しく接続されています。
>
> スイッチ Pin 3 (C) の no_connect: 未追加。上側ポールのPin 3にはまだno_connectマーカーが付いていません。

KiCadの回路図は座標ベースのテキストデータなので、AIが修正前後の差分を読み取れる。「修正しました」→「確認ください」のサイクルを何度も回せるのが、この協業スタイルの利点。

## Phase 3: JLCPCB部品選定

### AIにできたこと

- JLCPCBの部品ページ（URL）を渡すと、スペック・在庫・価格を読み取って要件との適合を判断
- 部品種別ごとの選定ポイント（USB-Cのピン数、コネクタの極性、パッケージサイズ等）を助言
- Basic/Extended部品の違いとコストへの影響を説明
- EOL部品（廃番）の代替品探し

### 実際のやりとり

> **自分:** USB-Cコネクタ選定のポイントはなんだろうか
>
> **AI:** 回路図ではUSB-Cから使っているのは VBUS、GND、CC1/CC2（5.1Kプルダウン）だけ。D+/D-は使っていない。つまり電源供給専用なので、最小限のピンがあれば足ります。

> **自分:** https://jlcpcb.com/partdetail/CK-KMR221GLFS/C72443 こんなんでどうだろうか
>
> **AI:** C&K製で品質は良いですが、1個$0.51〜$0.62とかなり高いです。RST + BOOTで2個必要なので$1以上になります。RST/BOOTは開発時にしか使わないスイッチなので、安いもので十分とも言えます。どうしますか？

JLCPCBの部品ページのURLを貼ると、AIがページを取得してスペック・在庫・価格を読み取り、回路の要件との適合を判断してくれる。自分で部品を探し、AIに「これどう？」と聞くスタイルが効率的だった。

### 注意が必要だったこと

**JLCPCB部品の在庫は流動的。** 選定時に在庫があっても、発注時には0になっていることが複数回あった。タクトスイッチは3回変更することになった：

1. E-Switch TL3301AF160QG (6x6mm) → 在庫切れ
2. HX-3x6x2.5-X (3x6mm) → これも在庫切れ
3. Alps Alpine SKRPACE010 (4.2x3.2mm) → 在庫あり

部品変更のたびにフットプリント再生成が必要になるので、時間のロスが大きい。可能なら、在庫が潤沢な部品（数万個以上）を選ぶか、Basic partを優先するのが良い。

## Phase 4: フットプリント割り当て

### easyeda2kicad — 重要ツール

JLCPCBの部品にはEasyEDA形式のフットプリントが用意されているが、KiCadでは直接使えない。`easyeda2kicad` というPythonツールで変換できる：

```bash
pip install easyeda2kicad
easyeda2kicad --full --lcsc_id C139797 --output ./custom_footprints/SKRPACE010
```

LCSC番号を指定するだけで、シンボル・フットプリント・3Dモデルが生成される。EasyEDAのアカウント登録は不要。

### パッド名の不一致問題

生成されたフットプリントのパッド名が、KiCadの回路図シンボルのピン名と一致しないことがある。これは手動で修正が必要：

- **タクトスイッチ（4パッド → 2ピン）:** 内部で繋がっているパッド同士を同じピン番号にリネーム。**必ずデータシートで内部接続を確認すること。**
- **USB-C:** シールドパッド「7」→「S1」にリネーム

### 実際のやりとり

> **自分:** EasyEDAという画面が開いたが... ダウンロードできるのだろうか。Registerする必要がある？
>
> **AI:** もっと簡単な方法があります。easyeda2kicad コマンドラインツール -- アカウント不要で、JLCPCB部品番号からKiCadフットプリントを直接生成できます。
> ```
> pip install easyeda2kicad
> easyeda2kicad --lcsc_id C49023767 --output ./custom_footprints/MST-12D18G4
> ```

> **自分:** SW, USB-Cをフットプリント割り当てしたが、SWもピン表記が違うよね。きっと。
>
> **AI:** パッド名を確認しました。タクトスイッチ -- KiCadの SW_Push シンボルはピン2つ（1と2）しかありません。フットプリントは4パッドです。左の2パッド → 両方とも "1" に、右の2パッド → 両方とも "2" に。

AIはフットプリントファイル（.kicad_mod）も直接読めるので、パッド名の不一致を検出できる。ただし、どのパッド同士が内部接続されているかはデータシートの確認が必須。実際、自分がデータシートを見ずにリネームしてしまい、やり直すことになった。

## Phase 5: PCBレイアウト

PCBレイアウト自体はKiCadのPCBエディタで手動で行った。AIに聞いて役立った知識：

- **GND戦略:** B.Cu（裏面）をGNDベタにして、デカップリングコンデンサのGNDパッドからビアで落とす
- **ビア配置:** GNDパッドの近くに配置するほどインピーダンスが下がる
- **シルクスクリーン:** F.SilkSは基板に印刷される、F.Fabは印刷されない
- **ネジ穴:** `MountingHole` フットプリントを使用、30mm基板ならM2サイズ

### 実際のやりとり

> **自分:** 配線で、GNDはどうするのが良いかな
>
> **AI:** 基本方針は裏面（B.Cu）をGNDベタプレーンにすることです。GNDの配線は引かない -- 各部品のGNDパッドにビアだけ打ちます。

> **自分:** GNDピアと端子の間は狭い方が良い？
>
> **AI:** はい、狭い方が良いです。特に重要なのはデカップリングコンデンサのGNDパッドです。ここの配線が長いと、コンデンサの効果が薄れます。理想はコンデンサのGNDパッドから 0.3〜0.5mm の距離にビアです。

PCBレイアウトのGUI操作はAIにはできないが、「何をどう配置すべきか」「なぜそうすべきか」の知識は正確に提供してくれる。手を動かすのは自分、判断の根拠はAIに聞く、という分担。

## Phase 6: DRC/ERC

KiCadのDRC/ERCレポート（`.rpt`ファイル）をAIに読ませると、各エラーの意味と対処法を日本語で解説してくれる。

主な問題と対処：

| エラー | 原因 | 対処 |
|--------|------|------|
| annular_width | スイッチ固定穴のパッド径 < ドリル径 | PTH → NPTHに変更 |
| power_pin_not_driven | 電源入力ピンの駆動元がない | PWR_FLAGシンボルを追加 |
| unconnected_items | 配線未完了 | 配線を完了 |
| silk_edge_clearance | シルクが基板端からはみ出し | cosmetic、無視可 |

### 実際のやりとり

> **自分:** 自分で直せそうな奴は直したのだが、20260301 ってreportファイルを保存した
>
> **AI:** *(レポートファイルを読んで)* スライドスイッチ(Power)の固定穴 x2 -- アニュラー幅が-0.004mmになっている。穴に対してパッドが小さすぎます。

> **自分:** 電気的に不要 と思っていいですよね。
>
> **AI:** はい、その通りです。Power スライドスイッチの2つの取り付け穴は機械的な固定専用で、電気的には不要です。PTH → NPTHに変更すれば、annular_widthエラーとpadstack警告の計4件が一度に解消します。

KiCadのDRC/ERCが出力する .rpt ファイル（日本語）をそのままAIに渡すと、各エラーの意味を噛み砕いて説明し、具体的な修正手順を示してくれる。

**回路図↔PCBの整合性チェック** はPCBエディタのF8（回路図から基板を更新）で「変更なし」が出れば一致している。

## Phase 7: 製造ファイル生成

### Fabrication Toolkit（KiCadプラグイン）

KiCadのPlugin & Content ManagerからインストールできるFabrication Toolkitが便利。ワンクリックでJLCPCB向けのGerber、BOM、CPLファイルを生成する。

### 落とし穴

1. **LCSCフィールドが空になる:** 回路図でLCSCフィールドを追加した後、PCBエディタでF8して同期しないとBOMに反映されない
2. **DNPとExclude from BOMは別:** 回路図上の×マーク（DNP）だけではBOMから除外されない。「Exclude from BOM」チェックが別途必要
3. **production/backups/ が生成される:** .gitignoreに追加しておくのが良い

### 実際のやりとり

> **自分:** OK, production ディレクトリができた。どうだろうか
>
> **AI:** *(BOM CSVを読んで)* ファイル生成は成功しています。2点修正が必要です。LCSC Part # が全て空です。DISPLAY（J2）が BOM に含まれています。

> **自分:** LCSCの項目に、列チェックが入っていなかったので、入れ直してみました
>
> **AI:** *(再度読んで)* まだ空ですね。おそらく回路図の変更がPCBに反映されていないのが原因です。Fabrication Toolkit はPCBファイルから BOM を生成するので、回路図で追加した LCSC フィールドをPCBに同期する必要があります。PCBエディタで F8 → 適用 → 再度 Generate してください。

AIが生成されたCSVファイルを直接読んでチェックしてくれるので、「生成→確認→修正→再生成」のサイクルを素早く回せた。LCSCフィールドがPCBに同期されないというKiCad特有の落とし穴も、AIの指摘で解決できた。

## Phase 8: JLCPCB発注

### 発注の流れ

1. jlcpcb.com で Gerber ZIP をアップロード → PCB仕様を選択
2. PCBAを有効化 → BOM CSV と CPL CSV をアップロード
3. 部品マッチング確認 → コンポーネント配置プレビュー → 発注

### 「Confirm Parts Placement」はYesにすべき

JLCPCBのエンジニアが部品配置をレビューしてくれる。追加費用なし。今回、**これがSTM32のLCSC番号誤りを検出してくれた。**

### LCSC番号の取り違え — 最大の失敗

BOMに記載したU1のLCSC番号 C526973 は、実はSTM32G030F6Pxではなく**10pF 0402コンデンサ**だった。JLCPCBのエンジニアが「パッケージが合わない」と連絡してきて発覚。

**対策:** 発注前に、全LCSC番号をJLCPCBの部品ページで照合すべき。AIに頼んで全件検証したところ、この1件以外は全て正しかった。番号の出典が不確かなものは特に注意。

### 実際のやりとり

> **自分:** *(JLCPCBのエンジニア Doris からのメールを貼り付けて)*
> "seems the packages of the parts (U1) cannot match the corresponding pads on the PCB board, so we are unable to assemble them."
>
> **AI:** 原因がわかりました！ C526973 は STM32G030F6Px ではなく、0402 コンデンサ (10pF) です。LCSC 番号が間違っています。正しい番号は C724040 です。

> **自分:** ありがとう。念の為、BOMをチェックしてもらっていいですか。他に取り違えがあるのなら知っておきたい
>
> **AI:** *(全19部品のLCSC番号をJLCPCBのWebページで1件ずつ照合して)* 全19部品の検証が完了しました。問題は U1 の1件のみです。

> **自分:** ありがとう、迅速な対応への感謝をメッセージに入れたい。
>
> **AI:** "Hi Doris, thank you for catching this so quickly! Please go with option A. The correct part for U1 should be LCSC# C724040 (STM32G030F6P6, TSSOP-20). Thank you for your help!"

LCSC番号の問題が発覚した際、AIがWebページを取得して番号の実体を調べ、正しい番号を検索し、さらにBOM全件を1つずつ照合してくれた。英語の返信文も起草してくれる。Confirm Parts Placementを有効にしていたおかげで、製造前にこの問題をキャッチできた。

### コスト構造（5枚PCBA時）

| 項目 | 金額 |
|------|------|
| 固定費（設定・ステンシル・フィーダー等） | ~¥10,275 |
| 変動費（1枚あたり） | ~¥489 |
| **5枚合計** | **~¥12,247** |

固定費が支配的なので、枚数を増やすほど単価は下がる。¥1,000/枚を切るには約20枚必要。

## AIの活用まとめ

### 特に有効だった場面

| 場面 | AIの役割 |
|------|---------|
| 回路図レビュー | S式テキストを直接パースして全接続を検証 |
| 部品選定 | JLCPCBの部品ページを読み取り、スペック適合を判断 |

### AIにできなかったこと

- **KiCad GUIの直接操作:** フットプリント割り当て、パッドリネーム、PCBレイアウト、配線は全て手動
- **部品の実物確認:** データシートの物理寸法の最終判断は人間が行う必要がある
- **JLCPCBの在庫リアルタイム確認:** Web上の在庫とPCBA在庫は別管理の場合があり、正確な判断は発注画面でしかできない

### 使用したツール

| ツール | 用途 |
|--------|------|
| KiCad 9 | 回路図・PCB設計 |
| easyeda2kicad (pip) | JLCPCB部品のKiCadフットプリント変換 |

## 次にやるなら気をつけること

1. **LCSC番号は必ず照合する。** 手入力ミスは起きる。発注前に全件検証を。
2. **在庫の多い部品を選ぶ。** 数百個レベルの在庫は発注時に0になりうる。Basic partを優先。
3. **Confirm Parts Placementは必ずYes。** 無料で、致命的ミスのセーフティネットになる。
4. **easyeda2kicadのパッド名は信用しない。** データシートと突き合わせてリネームが必要。
5. **KiCadのLCSCフィールド追加後はF8で同期。** 忘れるとBOMに反映されない。
